## 1단계: 기업-연도 문서 만들기

수집한 문장들은 기업-연도 단위로 하나씩 묶습니다. 예를 들어 아래 세 문장을 뽑았다고 합시다.

| passage                             | 사람이 읽는 ESG 영역 |
| ----------------------------------- | ------------- |
| 당사는 온실가스 배출량을 감축하고 재생에너지 사용을 확대하였다. | 환경 E          |
| 협력사의 안전보건 교육을 강화하고 공급망 점검을 확대하였다.   | 사회 S          |
| 이사회 내 감사위원회의 독립성과 내부통제 절차를 강화하였다.   | 지배구조 G        |

학생 기본 분석에서는 한 기업-연도의 ESG 문장을 하나로 묶어 처리합니다.

```text
예시기업-2024 firm-year document (기업-연도 문서)
= 당사는 온실가스 배출량을 감축하고 재생에너지 사용을 확대하였다.
  협력사의 안전보건 교육을 강화하고 공급망 점검을 확대하였다.
  이사회 내 감사위원회의 독립성과 내부통제 절차를 강화하였다.
```

전체 수집 대상은 381개 기업-연도입니다. 다만 분석 corpus는 그중 사업보고서와 텍스트를 성공적으로 수집·추출한 기업-연도 문서로 만듭니다.

```text
len(corpus) = N   # corpus(말뭉치): 수집 성공 기업-연도 문서 수 (최대 381)
```

기업마다 문장 수는 다를 수 있습니다. 중요한 점은 회귀분석 표에서 기업-연도당 1행으로 집계한다는 것입니다.

> [!question] Decision box — 문장 묶기 기준
>- **선택지**: 보고서 전체를 하나로 묶기 / II·IV·VI 섹션만 쓰기 / E·S·G 문장을 따로 묶기  
>- **선택 기준**: 잡음을 얼마나 줄일 것인가, 출처 섹션을 얼마나 해석에 활용할 것인가  
>- **보고서에 쓸 판단**: "우리 팀은 ___ 기준으로 문장을 묶었다. 이 선택은 ___ 장점이 있지만, ___ 문장을 놓칠 수 있다."

## 2단계: TF-IDF 점수 만들기

> `data/stopwords_ko_esg.txt`에 불용어 초안이 있습니다. "및", "관련", "통해" 같은
> 일반 조사·부사를 제거한 출발점입니다. 팀 데이터에 맞게 직접 검토하고 보완하세요.
> (회사명, 연도 표현, 반복 법률 용어 등이 추가 후보입니다.)

TF-IDF(단어 빈도-역문서 빈도)는 "이 단어가 이 문서에서 얼마나 독특하게 자주 등장하는가"를 숫자로 나타냅니다. 아래 표는 이해를 위한 작은 예시입니다.

| term | TF-IDF |
|---|---:|
| 온실가스 | 0.20 |
| 재생에너지 | 0.18 |
| 배출량 | 0.12 |
| 감축 | 0.10 |
| 안전보건 | 0.15 |
| 공급망 | 0.08 |
| 감사위원회 | 0.12 |
| 이사회 | 0.09 |
| 사용 | 0.03 |

seed dictionary(씨앗 단어 목록)는 TF-IDF matrix(단어별 점수 행렬)를 만든 뒤, 관심 있는 단어 열만 고르는 기준입니다.

```text
E seed = [온실가스, 재생에너지, 탄소]
E seed score = 0.20 + 0.18 + 0
             = 0.38

S seed = [안전보건, 공급망, 근로자]
S seed score = 0.15 + 0.08 + 0
             = 0.23

G seed = [감사위원회, 이사회, 내부통제]
G seed score = 0.12 + 0.09 + 0
             = 0.21
```

expanded dictionary(확장 사전)는 E/S/G별 seed 단어 10개씩을 출발점으로 삼아, word embedding(단어 임베딩, FastText/Word2Vec)으로 후보를 넓힌 뒤, 팀이 세운 기준으로 걸러낸 단어 목록입니다. 
- **식은 seed score와 동일 — 합산할 단어 집합만 커집니다.** 
- 이미 정해진 "정답 확장 사전"을 그대로 쓰는 과제가 아니라, 
- 팀이 선택한 embedding·threshold·직접 검토·걸러내기 기준을 보고서에 설명해야 합니다.

확장 메커니즘 (작은 5차원 toy 예시):

```text
seed = [온실가스, 재생에너지]

각 seed 단어를 벡터로 변환 (실제 100-300차원, 여기는 5차원 toy):
v(온실가스)   = [0.42, -0.18, 0.55, 0.27, -0.31]
v(재생에너지) = [0.39,  0.05, 0.51, 0.30, -0.28]

말뭉치(corpus) 안의 다른 단어 벡터들과 cosine similarity 계산:

word            cos(v, 온실가스)   cos(v, 재생에너지)    max
탄소중립             0.92               0.88              0.92
배출권               0.88               0.71              0.88
저탄소               0.85               0.79              0.85
태양광               0.61               0.83              0.83
대림에너지           0.78               0.74              0.78   ← 회사명 잡음
경영                  0.31               0.28              0.31

threshold θ = 0.85 적용:
expanded_E(θ=0.85) = {온실가스, 재생에너지, 탄소중립, 배출권, 저탄소}
  - "대림에너지"(0.78) → threshold 미달로 제외
  - "경영"(0.31)         → 의미 무관으로 제외
```

→ **보고서 판단**: θ를 몇 가지 값으로 sweep하고, 확장 사전 크기와 잡음 단어 비율을 표로 보여준 뒤 자기 θ를 정당화합니다. 
- θ가 낮으면 후보 단어가 수백 개로 늘어나므로, 각 θ별로는 cosine 점수 기준 상위 100단어를 눈으로 검토해도 충분합니다. 
- 최종 선택한 θ에서 회사명·일반명사 같은 잡음 단어를 직접 검토해 걸러냈는지 기록합니다(Bao et al. 2024).

확장 사전이 정해졌으면, seed 식과 동일하게 TF-IDF를 합산합니다:

```text
E expanded = [온실가스, 재생에너지, 탄소, 배출량, 감축]
E expanded score = 0.20 + 0.18 + 0 + 0.12 + 0.10
                 = 0.60
```

두 점수는 기본 분석에서는 각각 별도 모델로 비교합니다. **동시에 넣으면 다중공선성 때문에 계수 해석이 불안정해질 수 있어요.**

> **방법론 기반 — firm-year → E score (Li et al. 2021)** — 위 toy 예시가 보여주는 핵심 흐름:
>
> ```text
> 예시기업-2024 (firm-year 1개)
> ↓ DART II·IV·VI 섹션 ESG 문장 수집 → 하나의 문서로 묶기
> ↓ 수집·추출에 성공한 firm-year 문서 기준으로 TF-IDF fit
> ↓ E seed 단어 TF-IDF 값 합산
> E_score = TF-IDF("온실가스") + TF-IDF("재생에너지") + ... = 0.38
> S_score = TF-IDF("안전보건") + TF-IDF("공급망")    + ... = 0.23
> G_score = TF-IDF("감사위원회") + TF-IDF("이사회")  + ... = 0.21
> ```
>
> 성공적으로 수집·추출한 firm-year 전체에 이 절차를 반복하면 회귀분석 입력 표가 완성됩니다. 실패하거나 누락된 행은 최종 보고서에 이유를 적고, 가짜 0으로 채워 분석 corpus에 넣지 않습니다. 이 방식이 Li et al. (2021, *Review of Financial Studies*)의 핵심이고, E/S/G 차원별 점수를 각각 회귀하는 방식은 Bai et al. (2024)에서 사용합니다 — `references/02_key_literature.md` 참고.

실행 흐름은 길게 외우는 절차가 아니라, 노트북에서 직접 구현하고 판단해야 할 흐름입니다.

```text
1. CSV를 불러온다.
2. firm-year 단위로 ESG passage를 하나의 text로 묶는다.
3. seed dictionary와 expanded dictionary를 각각 TF-IDF 점수로 만든다.
4. ESG grade를 숫자로 매핑한다.
5. M1a(seed only)와 M1b(expanded only)를 비교한다.
```


> [!question] Decision box — 사전 확장과 threshold
>- **선택지**: seed-only / expanded dictionary / fastText 확장 단어 빈도  
>- **선택 기준**: 해석 가능성, 후보 단어의 잡음, KCGS 등급과의 검증 결과  
>- **보고서에 쓸 판단**: "θ를 ___로 정한 이유는 ___이다. 낮은 θ에서는 ___ 잡음이 늘었고, 높은 θ에서는 ___ 단어를 놓쳤다."

## 3단계: feature가 등급과 연결되는지 먼저 확인하기

회귀분석으로 바로 가기 전에, 만든 feature가 ESG 등급과 최소한의 통계적 연관을 보이는지 확인합니다. 이때 Pearson correlation보다 **Spearman 순위상관**을 먼저 봅니다.

Spearman은 숫자의 크기 자체보다 **순위가 같이 움직이는지**를 봅니다. ESG 등급은 D, C, B, B+, A, A+처럼 간격이 정확히 같은 숫자가 아니라 순서가 중요한 서열변수이므로, "텍스트 점수가 높은 기업이 대체로 더 높은 등급 순위에 놓이는가?"를 보는 Spearman이 자연스럽습니다.

작은 예시:

| firm-year | ESG text score | ESG grade numeric | text score rank | grade rank |
| --------- | -------------: | ----------------: | --------------: | ---------: |
| A         |           0.10 |                 2 |               1 |          1 |
| B         |           0.30 |                 3 |               2 |          2 |
| C         |           0.80 |                 5 |               4 |          4 |
| D         |           0.50 |                 4 |               3 |          3 |

이 경우 text score가 높을수록 grade 순위도 같이 올라가므로 Spearman은 양수에 가깝습니다. 반대로 text score는 높은데 등급 순위가 낮으면 Spearman은 0에 가까워지거나 음수가 됩니다.

보고서에는 아래처럼 feature 검증표를 먼저 넣는 것이 좋습니다.

| feature | grade | Spearman ρ | p-value | 조심스러운 해석 |
|---|---|---:|---:|---|
| expanded_tfidf_score | ESG grade | 0.15 | 0.004 | 텍스트 점수가 높은 기업-연도일수록 등급도 높은 경향은 있으나, 관계는 약함 |
| seed_tfidf_score | ESG grade | 0.03 | 0.55 | 등급과 뛚렬한 순위 관계를 보이지 않음 |

ρ의 절댓값이 작고 p-value가 크면, 그 feature는 회귀에 넣기 전에 "왜 약한가"를 먼저 진단해야 합니다. 단, p-value가 작아도 ρ가 0.10-0.20 정도라면 강한 예측력이 아니라 **약한 연관성**으로 해석해야 합니다.

Spearman만으로 충분하지는 않습니다. 
- 높은 등급 그룹과 낮은 등급 그룹의 feature 평균이 다른지도 t-test 또는 Mann-Whitney test로 함께 확인하면, 
- feature가 등급과 연결되는 방식을 더 구체적으로 설명할 수 있습니다.

> [!question] Decision box — feature 검증
>- **선택지**: Spearman만 보고 / Spearman + 평균 차이 검정 / 문장 예시까지 함께 읽기  
>- **선택 기준**: 등급이 서열변수라는 점, feature 분포의 치우침, 표본 수  
>- **보고서에 쓸 판단**: "이 feature는 ___와 통계적으로 연결되지만, 효과 크기는 ___ 수준이므로 ___로 해석한다."

## 4단계: 심화 분석 — 기준 문장 cosine

이 과제에서는 `esg_year = fiscal_year + 1`로 맞춥니다. `esg_year`는 KCGS 평가연도이며, 예를 들어 2025 ESG 등급은 2024.12 사업보고서 언어와 연결합니다.

등급 숫자 변환 기준:

```text
S=6, A+=5, A=4, B+=3, B=2, C=1, D=0
```

아래 E/S/G 기준 문장을 활용해 공시 텍스트와의 cosine similarity feature를 추가할 수 있습니다.

| 기준 | 기준 문장 |
| --- | ----------------------------------------- |
| E   | 온실가스 배출 감축, 재생에너지 사용, 탄소중립 목표와 실행 계획 |
| S   | 산업안전, 근로자 보호, 공급망 책임, 고객과 지역사회에 대한 사회적 책임 |
| G   | 이사회 독립성, 감사위원회, 내부통제, 주주권리와 윤리경영 |

> [!question] 심화 분석
>- 공시 텍스트와 기준 문장을 어떤 단위로 비교할지가 핵심 설계 결정입니다
>- 구현 방법과 그 이유를 보고서에 적으세요

## 5단계: 회귀분석으로 읽기

ESG 등급은 D~A+의 **서열변수(ordinal)** 입니다. 
- 등간성 가정(A+→A 차이 = B→C 차이)이 엄밀하게는 깨집니다. 
- 아래 모형 중 팀이 선택해 사용하고, 보고서에 선택 근거를 짧게 적습니다. 
- 중요한 것은 "어떤 모형이 무조건 정답인가"가 아니라, 등급 변수를 어떻게 해석했는지와 그 선택의 한계를 함께 쓰는 것입니다.

### 옵션 A: OLS (baseline, 직관 우선)

```text
esg_grade_num = β0 + β1·seed_tfidf_score + β2·total_word_count + ε
```

`total_word_count`는 **cheap-talk 통제 변수**입니다. β1이 word_count를 통제한 후에도 양수 + 유의하면, 텍스트 점수가 단순 verbosity(공시 장황함) 이상의 신호를 가진다는 증거가 됩니다.

장점: 회귀 해석이 직관적. 한계: 등간성 가정이 ordinal 등급에 부정확.

### 옵션 B: Ordered logit (등급 순서 보존)

```text
ordered_logit(esg_grade_num)
  = β1·seed_tfidf_score + β2·total_word_count + ...
```

D, C, B, B+, A, A+처럼 등급의 순서는 보존하되, 등급 간 간격이 모두 같다고 강하게 가정하지 않습니다. `statsmodels.miscmodels.ordinal_model.OrderedModel`을 사용할 수 있습니다.

장점: ESG 등급이 서열변수라는 성격에 가장 잘 맞음. 한계: OLS보다 계수 해석이 어렵고, 표본이 작거나 predictor가 많으면 결과가 불안정해질 수 있음.

### 옵션 C: Binary logit (상위 등급 여부)

```text
P(esg_grade ≥ A) = logistic(β0 + β1·seed_tfidf_score + β2·total_word_count)
```

A 이상(esg_grade_num ≥ 4)을 1, B+ 이하를 0으로 변환. `statsmodels.Logit` 한 줄. β1이 양수이면 텍스트 점수가 상위 등급(A+, A) 확률을 높이는 경향.

장점: A 이상 여부처럼 해석이 직관적이고 odds ratio를 설명하기 쉬움. 한계: A 이상/미만 두 그룹으로 압축해 grade 간 미묘한 차이를 잃음.

> **옵션 선택 정당화**: 보고서에는 "왜 OLS인가", "왜 ordered logit인가", 또는 "왜 binary logit인가"가 한 단락으로 드러나야 합니다. 예를 들어 OLS는 baseline 해석용, ordered logit은 등급 순서 보존용, binary logit은 A 이상 여부 질문에 맞춘 모형입니다.

해석 예시 (둘 다 적용):

| 결과 | 조심스러운 해석 |
|---|---|
| `seed_tfidf_score` β1 양수 + 유의 | seed 단어 강한 기업-연도일수록 ESG 등급 ↑ 경향 |
| `total_word_count` 통제 후 β1 유지 | 텍스트 신호가 verbosity 넘는 ESG 측정 가능성 |
| `total_word_count` 통제 후 β1 ≈ 0 | 신호의 상당 부분이 단순 disclosure 장황함 — cheap-talk 가설 강해짐 |
| `E 기준문장 cosine` 양수 + 유의 | 환경 관련 단어 겹침이 ESG 등급과 양상관 (baseline) |
| 부호 불안정 / 표본 작음 | feature 다중공선성 또는 표본 한계 |
