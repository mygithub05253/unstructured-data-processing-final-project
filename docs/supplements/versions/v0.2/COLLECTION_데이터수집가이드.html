웹 크롤링 개념 — 비전공자용 정리

Week 4 강의의 핵심을 이 프로젝트에 맞게 압축. 크롤링은 단순히 "긁어오기"가 아니라 여러 가지 방법 중 가장 적절한 것을 선택하는 일입니다.

1. Static vs Dynamic 사이트

📄 Static (정적) 사이트

서버가 완성된 HTML을 한 번에 내려줌. requests.get(url)으로 받으면 끝.

DART의 viewer URL이 여기. BeautifulSoup으로 파싱 가능.

⚙️ Dynamic (동적) 사이트

초기 HTML은 비어 있고, JavaScript가 추가로 데이터를 fetch해서 채움.

Selenium / Playwright 필요하지만, 이 프로젝트에선 API 직접 호출이 우선이라 거의 안 씀.

Static = 신문지를 통째로 받는 것 (글이 다 인쇄돼 있음). Dynamic = 신문이 백지로 오고, 옆에 작가가 손글씨로 글을 채워 넣는 것 (브라우저 = 작가). API 호출은 신문사 본사 창고에서 원문을 직접 받는 것 — 가장 빠르고 정확.

import requests

# GET 요청 — params로 query string 전달
res = requests.get(

    params={

        "corp_code": "00126380",

        "end_de": "20251231",
        "pblntf_detail_ty": "A001",  # 사업보고서
        "last_reprt_at": "Y",        # 정정 시 최신만
    },
    timeout=20,
)
res.raise_for_status()
payload = res.json()

4. robots.txt와 ToS

  • robots.txt 확인: https://opendart.fss.or.kr/robots.txt처럼 도메인 끝에 /robots.txt 붙이면 크롤링 허용 범위 확인 가능.
  • User-Agent 명시: headers={"User-Agent": "ESG-DART-research/0.1 (academic)"}로 정체 밝히기.
  • Rate limit: 요청 간 time.sleep(0.1) 정도. OpenDART는 분당 약 600회 제한 권장.
  • 캐시: 같은 URL을 반복 호출하지 말고 data/cache/에 저장 후 재사용.
윤리·법적 주의 공개 데이터라도 대량 크롤링은 서버 부하·법적 문제를 일으킬 수 있음. 학술 연구·소규모 표본(381건)에서는 보통 문제 없지만, "academic", 연락처, 목적을 User-Agent에 적는 게 안전.

주의 — list.json의 함정

⚠️ "사업보고서 (2024.12)" 정확히 매칭
  • list.json은 회사의 모든 공시를 반환. "사업보고서"라는 단어가 들어간 다른 공시(정정 등)도 섞임.
  • 매칭 규칙: report_nm에 "사업보고서" 포함 AND f"({fiscal_year}.12)" 포함
  • 정정공시가 여러 개면 last_reprt_at=Y로 가장 최신 행 1개만.
  • 최종 사용한 rcept_no를 collection_log에 기록.

섹션 추출 — II · IV · VI

DART 사업보고서 표준 목차

  • I. 회사의 개요 — 회사 일반 정보 (이번 분석 제외)
  • II. 사업의 내용 ✅ — 사업 영역, ESG 활동 설명 (E 신호 多)
  • III. 재무에 관한 사항 — 재무제표 (재무 통제변수용, 선택)
  • IV. 이사의 경영진단 및 분석의견 ✅ — 경영진 시각의 ESG 언급 (S·G 신호)
  • V. 회계감사인의 감사의견 등 — 일반적으로 제외
  • VI. 이사회 등 회사의 기관에 관한 사항 ✅ — 지배구조 (G 신호 多)
  • VII~XII — 주주, 임원, 계열사 등 (선택)

추출 전략: 헤딩 마커 정규식(예: r"II\.\s*사업의\s*내용") → 다음 헤딩까지 텍스트 슬라이스.

섹션 추출의 함정 — DART XML은 회사마다 형식이 약간씩 다름. 헤딩 라벨이 "Ⅱ"(로마자 II)인지 "II"(영문 II)인지, "사업의 내용"인지 "사업의내용"인지 변형이 있음. 여러 패턴을 OR로 매칭하고 추출 실패율을 모니터링.

기능 #1 — Rate Limit (요청 페이스)
import time
from collections import deque

class RateLimiter:
    """분당 N회 제한. OpenDART는 분당 ~600 권장."""
    def __init__(self, max_per_minute=300):
        self.max = max_per_minute
        self.window = 60
        self.calls = deque()

    def wait(self):
        now = time.monotonic()
        while self.calls and now - self.calls[0] > self.window:
            self.calls.popleft()
        if len(self.calls) >= self.max:
            sleep_for = self.window - (now - self.calls[0]) + 0.1
            time.sleep(sleep_for)
        self.calls.append(time.monotonic())
기능 #2 — Exponential Backoff (실패 시 재시도)
import time, random

def request_with_backoff(url, params, max_retries=5):
    """503·timeout 발생 시 지수 backoff로 재시도."""
    for attempt in range(max_retries):
        try:
            res = requests.get(url, params=params, timeout=20)
            if res.status_code == 200:
                return res
            if res.status_code in (429, 503):
                wait = (2 ** attempt) + random.uniform(0, 1)
                time.sleep(wait)
                continue
            res.raise_for_status()
        except requests.exceptions.Timeout:
            time.sleep(2 ** attempt)
    raise RuntimeError(f"Failed after {max_retries} retries: {url}")
기능 #3 — Idempotent Cache (재실행 안전)
from pathlib import Path

def get_or_fetch_document(rcept_no, api_key):
    """이미 받은 zip은 재다운로드 ❌."""
    cache_path = Path(f"data/raw/{rcept_no}.zip")
    if cache_path.exists() and cache_path.stat().st_size > 1000:
        return cache_path  # 이미 있음 → 즉시 반환

    res = request_with_backoff(
        "https://opendart.fss.or.kr/api/document.xml",
        params={"crtfc_key": api_key, "rcept_no": rcept_no},
    )
    cache_path.parent.mkdir(parents=True, exist_ok=True)
    cache_path.write_bytes(res.content)
    return cache_path

import pandas as pd
from pathlib import Path

LOG_PATH = Path("logs/collection_log.csv")

def load_done_set():
    """이미 처리된 firm-year set 반환."""
    if not LOG_PATH.exists():
        return set()
    df = pd.read_csv(LOG_PATH)
    return set(zip(df["stock_code"], df["fiscal_year"]))

def main():
    master = pd.read_csv("data/company_master.csv")
    done = load_done_set()
    for _, row in tqdm(master.iterrows(), total=len(master)):
        key = (row["stock_code"], row["fiscal_year"])
        if key in done:
            continue  # 재실행 시 건너뜀
        result = collect_one(row)
        append_log(result)
기능 #5 — Structured Logging
def append_log(stock_code, fiscal_year, corp_code, rcept_no,
               status, reason="", timestamp=None):
    row = {
        "stock_code": stock_code,
        "fiscal_year": fiscal_year,
        "corp_code": corp_code or "",
        "rcept_no": rcept_no or "",
        "status": status,             # SUCCESS / FAIL_CORP / FAIL_RCEPT / FAIL_DOC
        "reason": reason,
        "timestamp": timestamp or datetime.now().isoformat(timespec="seconds"),
    }
    pd.DataFrame([row]).to_csv(
        LOG_PATH,
        mode="a",
        header=not LOG_PATH.exists(),
        index=False,
    )
