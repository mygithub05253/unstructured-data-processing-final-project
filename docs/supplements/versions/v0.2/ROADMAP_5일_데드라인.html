5일 작전도 — 데이터가 분석의 천장이다

이 5일 동안 우리가 만드는 건 "분석에 쓸 corpus"가 아니라 분석이 가능해지는 corpus입니다. 회귀·알파 단계에서 결과가 안 나오면 거의 100% 이 5일 안에 흘려보낸 quality 문제로 회귀합니다.

이 5일은 건축의 기초 공사입니다. 5/20부터 시작될 회귀·시각화·발표는 그 위에 올리는 골조와 인테리어죠. 골조가 아무리 멋져도 기초가 얕으면 다 무너집니다. 그래서 오늘부터 5/19까지의 모든 시간은 "재료 quality"에 씁니다.

B → C 의존

  • B의 sections.json 없이는 C의 전처리 불가
  • 병목 방지: 5/16에 50건만 먼저 B → C 전달 → C가 코드 검증 → 5/17에 batch가 들어와도 즉시 처리 가능

  • C의 esg_features.parquet 없이는 D의 회귀 unblock 불가
  • 병목 방지: 5/18 오전에 C가 50건 sample로 esg_features 컬럼 스키마를 D에게 알려줌 → D는 schema 기준으로 join 코드 미리 작성

D → A 의존

  • D의 analytical_dataset이 게이트 통과해야 A의 FREEZE 선언 가능
  • 병목 방지: A는 5/19 게이트 12개 체크리스트를 5/18에 미리 정리. 회의 30분 안에 끝나도록

A → 전원 의존

  • A의 PR 머지/검토가 느리면 B·C·D 모두 stalled
  • 병목 방지: 본 5일 동안 A는 PR 평균 응답시간 4시간 이내 (slack ping 1시간)

모두가 매일 해야 할 것

  • 아침에 git pull origin main → 어제 머지된 게 무엇인지 확인
  • 오늘 task를 시작하기 전 본 ROADMAP의 해당 일자 카드 한 번 읽기
  • 점심에 (또는 점심 후) progress 1줄 슬랙/카톡 공유
  • 저녁에 그 날 lead task 산출물을 PR로 올림 (작아도 OK, 다음 날 reviewer 부담↓)
  • blocker 있으면 즉시 슬랙 (혼자 끙끙대지 않음)
