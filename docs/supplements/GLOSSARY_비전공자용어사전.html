A. 텍스트 처리 기초 — 비정형 데이터를 다루기 위한 5가지 기본

수업 Week 2-3에서 본 개념을 ESG DART 도메인에 맞춰 다시 풀었습니다. 형태소 분석기 4종은 발표 때 "왜 Okt(또는 Mecab)를 골랐나" 답할 무기입니다.

A-01 선택

Corpus / Token 말뭉치 / 토큰

① 한 줄 비유
Corpus = 도서관에 모은 책 더미, Token = 책을 자른 가장 작은 의미 조각. 도서관(corpus)이 없으면 분석할 게 없고, 자르는 방법(token)에 따라 같은 책이 다르게 읽힘.
② 작은 예시
Corpus = [
  "당사는 온실가스를 감축하였다",
  "협력사 안전보건 교육을 강화하였다",
  ... (성공 수집된 firm-year 문서 N개)
]
Token("당사는 온실가스를 감축하였다") =
  공백 분리 → ['당사는','온실가스를','감축하였다']
  형태소 분리 → ['당사','는','온실가스','를','감축','하','였','다']
③ 그림
전체 사업보고서 → ESG 문단 추출 → firm-year 문서 1개 = corpus의 1행
                                      ↓
                          토큰화: 문장 → 단어 리스트
④ 우리 연결
03 분석예시 md "기업-연도 문서" = corpus의 한 행. 최대 381개 firm-year를 모으되, 수집 실패행은 가짜 0이 아니라 corpus에서 제외. len(corpus) = N은 수집 성공 수.
A-02 필수

형태소 분석기 4종 Okt / Mecab / Komoran / Kkma

① 한 줄 비유
한국어를 "의미 단위로 자르는 칼 4종". 빵을 칼로 자르는 것과 식빵 슬라이서로 자르는 것이 다르듯, 같은 문장도 분석기마다 다르게 잘립니다.
② 작은 예시
입력: "당사는 온실가스 배출량을 감축하고 있다"
분석기출력특징
Okt당사 / 는 / 온실 / 가스 / 배출량 / 을 / 감축 / 하고 / 있다합성어 분해 경향, 신조어 강함
Mecab당사 / 는 / 온실가스 / 배출량 / 을 / 감축 / 하 / 고 / 있 / 다합성어 유지, 가장 빠름, Windows ❌
Komoran당사 / 는 / 온실가스 / 배출량 / 을 / 감축 / 하 / 고 / 있 / 다사용자 사전 추가 쉬움
Kkma당사 / 는 / 온실 / 가스 / 배출 / 량 / 을 / 감축 / 하 / 고 / 있 / 다분해가 가장 세밀, 가장 느림
③ 표 — 우리 도메인에서 무엇이 중요한가
판단 기준왜 ESG에서 중요한가
"온실가스" 유지"온실/가스"로 분해되면 seed 사전 매칭 안 됨 → Mecab/Komoran 유리
회사명 보존NER 단계에서 회사명을 제거해야 하므로 회사명을 한 덩어리로 잡아주는 분석기 유리
설치 난이도Windows 팀원 있으면 Mecab 설치 어려움 → Okt 권장
속도381 firm-year × 평균 수천 토큰. Kkma는 너무 느림
④ 우리 연결
심층가이드 결정박스 ③의 선택지. 강의 자료 UD-03-300, UD-03-301 연결. 발표 답안: "Windows 팀원이 있어 Okt를 선택. 신조어 강하고 설치 쉬워서. 단 '온실가스'가 분해되는 경향이 있어 사용자 사전 추가로 보완했다." 또는 "맥/리눅스 통일 가능해서 Mecab. 합성어 유지가 ESG 도메인 단어 매칭에 결정적."
A-03 권장

불용어 Stop words

① 한 줄 비유
모든 문서에 너무 자주 나와서 "있어도 그만 없어도 그만"인 단어. 도서관 책 색인에 "그리고/또한/다음과 같이"가 빠져있는 것과 같음.
② 작은 예시
일반 불용어: 및 / 관련 / 통해 / 한 / 또한 / 같이 / 위해 / 이에 / 더불어
제거 전: "당사는 ESG 경영을 통해 지속가능 성장을 위해 노력한다" (10 토큰)
제거 후: "당사 ESG 경영 지속가능 성장 노력" (6 토큰) — 의미 중심 단어만 남음
③ 그림
전처리 전:  [당사, 는, ESG, 경영, 을, 통해, ...]
              ↓ 불용어 사전 매칭
전처리 후:  [당사, ESG, 경영, ...]   ← TF-IDF에서 신호 강해짐
④ 우리 연결
data/stopwords_ko_esg.txt가 출발점. 불용어가 부족하면 "통해", "관련" 같은 단어가 상위 TF-IDF를 차지해 ESG 단어 신호가 묻힘. 자세한 추가 후보는 A-05(도메인 불용어) 참고.
A-04 권장

NER (개체명 인식) Named Entity Recognition

① 한 줄 비유
"사람·회사·지명 같은 고유명사를 기계가 자동으로 골라내는 작업". 신문 기사에서 인명·지명·기업명에 노란 형광펜 치는 것과 같음.
② 작은 예시
입력 : "삼성전자는 2024년 재생에너지 사용을 확대했다"
NER : [삼성전자=ORG, 2024년=DATE]
제거 후: "재생에너지 사용 확대"
③ 그림 — 왜 우리에게 결정적인가
NER 안 됐을 때:
  seed: "재생에너지"
  fastText 유사어 후보:
    탄소중립 (0.92) ✓
    저탄소   (0.85) ✓
    대림에너지(0.78) ✗ ← 회사명이 confusing 후보로 등장!

NER 됐을 때:
  "대림에너지"는 ORG 태그 받아 corpus에서 제거됨
  → 후보 목록에 아예 안 올라옴
④ 우리 연결
심층가이드 단계 3·5에서 핵심. NER이 안 되면 사전 확장 단계에서 회사명이 ESG 단어로 잡혀버림. Li 2021도 spaCy NER로 organization 제거. 우리는 한국어 NER 라이브러리가 부족하므로 대안: company_master.csv의 회사명 127개를 회사명 불용어 사전으로 자동 생성.
A-05 권장

도메인 불용어 Domain stop words

① 한 줄 비유
"이 분야 문서에서만 너무 자주 나오는 단어". 학교 문서의 "학생", 병원 문서의 "환자"처럼 도메인 안에서 무의미하게 반복되는 단어. 일반 불용어 사전엔 없음.
② 작은 예시 — DART 사업보고서 도메인
범주예시왜 제거
회사명삼성전자, 현대자동차, 우리은행특정 회사명이 ESG 단어로 잘못 매칭 (A-04 참고)
연도/날짜2024년, 2024.12, 당기, 전기, 당해 사업연도모든 보고서에 나옴 → 정보량 0
법률 정형구본 보고서, 당사는, 이하 같음, 다음과 같이보일러플레이트, ESG와 무관
표/각주 잔재단위, 주1), (*), 백만원, 천원표에서 XML 추출 시 들어옴
③ 그림 — 자동 생성 방법
company_master.csv → company_name 컬럼 → 127개 회사명
                            ↓
                      stopwords_ko_esg.txt + 회사명 + 자회사명·약칭 = 우리 팀 도메인 불용어
                            ↓
                      형태소 분석 후 토큰별 매칭 제거
④ 우리 연결
심층가이드 단계 3 "불용어" 카드에 명시. 전처리 단계에서 가장 학생 차별화 되는 지점. 회사명 자동 생성을 안 하면 점수가 회사명에 묶여 오염됨. Bao 2024와 Li 2021 모두 domain stop word 비중을 크게 잡음.

B. 텍스트 → 숫자 — TF-IDF가 무엇을 측정하는가

텍스트를 숫자로 바꾸는 가장 표준적인 방법. 비전공자에게 가장 중요한 건 "TF-IDF 점수가 높다 = 그 문서에서만 유독 자주 나오는 단어"라는 직관입니다.

B-06 선택

TF (Term Frequency) 단어 빈도

① 한 줄 비유
"이 문서에서 이 단어가 몇 번 나왔는가". 가장 단순한 카운트.
② 작은 예시
문서 A: "온실가스 감축 온실가스 절감 노력" → TF(온실가스) = 2, TF(감축) = 1
③ 그림
TF 단점: 길이가 긴 문서가 무조건 점수↑
  문서 A (100단어): "온실가스" 5번 → TF=5
  문서 B (1000단어): "온실가스" 10번 → TF=10
  → B가 더 ESG 강한 것처럼 보이지만, 비율로 보면 5%(A) vs 1%(B)
④ 우리 연결
TF만 쓰면 사업보고서 길이에 좌우됨. 그래서 IDF(B-07)와 곱해서 TF-IDF로 보정합니다. cheap-talk 가설(F-25)이 여기서 출발.
B-07 필수

IDF (Inverse Document Frequency) 역문서 빈도

① 한 줄 비유
"이 단어가 얼마나 특별한가"를 측정. 모든 문서에 다 나오면 흔한 단어(IDF↓), 소수 문서에만 나오면 특별한 단어(IDF↑).
② 작은 예시
전체 corpus = 100개 firm-year 문서
  "당사" → 100개 모두에 등장 → 흔함 → IDF ≈ 1.0
  "이사회" → 90개에 등장      → 흔함 → IDF ≈ 1.1
  "넷제로" → 20개에만 등장   → 특별 → IDF ≈ 2.6
  "GHG"   → 5개에만 등장     → 매우특별 → IDF ≈ 4.0

공식 (sklearn 기본): IDF = log((1+N) / (1+df)) + 1
  N = 전체 문서 수, df = 그 단어가 등장한 문서 수
③ 그림
모든 문서에 나옴  ─────────────→ 1.0 (있으나마나)
                       IDF
소수 문서에만   ─────────────→ 3-4 (특별한 신호)
④ 우리 연결
"당사", "보고서" 같은 보일러플레이트는 IDF가 낮아 TF-IDF에서 자동 감점됨 → 불용어로 다 못 잡아도 IDF가 일부 보정. 단 회사명은 IDF로 안 잡힘(특정 회사 문서에만 자주 나옴 = 특별해 보임) → A-04 NER 필요.
B-08 필수

TF-IDF Term Frequency × Inverse Document Frequency

① 한 줄 비유
"이 문서에서 자주(TF) 나오면서 다른 문서엔 잘 안 나오는(IDF) 단어"에 높은 점수. 즉 "이 문서를 특징짓는 단어".
② 작은 예시
삼성전자 2024 firm-year 문서:
  단어         TF    IDF    TF-IDF
  "당사"        20    1.0    0.20    ← 흔하니 가중치 낮음
  "온실가스"    5     2.5    0.50    ← 특별하면서 자주
  "넷제로"      3     3.5    1.05    ← 매우 특별 + 적당히 나옴 = 강한 신호
  "재생에너지"  4     2.8    0.56
③ 그림 — 우리 점수 계산 흐름
firm-year 문서 → 토큰화 → TF-IDF 행렬 (문서 × 단어)
                                    ↓
                          seed 단어 열만 선택해 합산
                                    ↓
                            seed_tfidf_score
                          (E·S·G 합 또는 차원별)
④ 우리 연결
03 분석예시 md "TF-IDF 점수 만들기" 직접. E seed score = 0.20 + 0.18 + 0 = 0.38처럼 단어별 TF-IDF를 단순 합산. 발표 답안: "TF-IDF는 '이 firm-year만 유독 ESG 단어를 강조했는지'를 측정. 단순 빈도(TF)만 쓰면 사업보고서 길이에 좌우되지만, IDF가 흔한 단어 가중치를 낮춰 신호를 정제."
B-09 권장

n-gram unigram / bigram / trigram

① 한 줄 비유
"단어를 몇 개씩 묶어서 볼 것인가". 단어 1개씩 = unigram, 2개 연속 묶음 = bigram, 3개 = trigram.
② 작은 예시
입력: "재생에너지 사용 확대"
  unigram (1)      : [재생에너지, 사용, 확대]
  bigram  (2)      : [재생에너지 사용, 사용 확대]
  unigram+bigram  : 둘 다 합쳐서 사용

왜 bigram이 필요한가:
  "안 좋다" — unigram만 보면 ["안", "좋다"] → "좋다"가 따로 보여서 POSITIVE처럼 보임
  bigram 추가 → ["안 좋다"] 패턴이 별도 feature → NEGATIVE 캡처
③ 표 — ESG에서의 효과
n-gram예시 패턴장점
unigram온실가스, 감축, 재생에너지기본 신호
bigram온실가스 감축, 재생에너지 확대"X 감축 / X 확대" 같은 행동 표현 캡처
④ 우리 연결
가이드는 unigram만 기본으로 가정하지만, 알파 분석 후보로 bigram 추가 실험 가치 있음. sklearn TfidfVectorizer(ngram_range=(1,2))로 한 줄.
B-10 선택

Vector 벡터

① 한 줄 비유
"단어나 문서를 숫자 좌표로 표현한 것". 2차원이면 (x, y), 100차원이면 (x1, x2, ..., x100). 좌표가 가까우면 의미도 가깝다고 본다.
② 작은 예시
TF-IDF 벡터 (단어 차원이 vocab 크기):
  문서 A = [0, 0.2, 0, 0.5, ..., 0, 0.1]   ← 대부분 0 (sparse)
                ↑          ↑
              "당사"   "온실가스"

Word embedding 벡터 (보통 100-300차원):
  v(온실가스)   = [0.42, -0.18, 0.55, 0.27, -0.31, ...]
  v(재생에너지) = [0.39,  0.05, 0.51, 0.30, -0.28, ...]
  → 좌표가 비슷 → cosine similarity 높음
③ 그림
2차원 상상도:
          y
          |
       온실가스 *
          |  * 재생에너지
          |
          | * 탄소중립
          |____________________
                                  x
        "회의" ↑ 와 "온실가스" 사이 거리 멀음
        "재생에너지" ↑ 와 "온실가스" 거리 가까움
④ 우리 연결
TF-IDF 벡터는 sparse(대부분 0). word embedding 벡터는 dense(대부분 의미 있는 값). 둘 다 우리 프로젝트에 등장. cosine similarity(C-13)는 이 두 벡터 형식 모두에 적용 가능.

C. 단어 임베딩 + 사전 확장 — 우리 프로젝트의 학술적 핵심

이 카테고리가 발표 시 교수 질문의 50%를 차지합니다. 특히 θ sweep의 정당화(결정박스 ⑤)는 발표 핵심. 5개 모두 필수.

C-11 필수

Word Embedding 단어 임베딩

① 한 줄 비유
"의미가 비슷한 단어를 가까운 좌표에 배치하는 지도 만들기". 컴퓨터가 "온실가스"와 "탄소"를 보고 "이 둘은 비슷한 자리에 있구나"를 학습.
② 작은 예시
학습 전: 단어들이 컴퓨터에겐 그냥 다른 문자열
학습 후: 비슷한 맥락에 등장한 단어 → 가까운 좌표

  v(온실가스)   = [0.42, -0.18, 0.55, 0.27, -0.31]   ┐
  v(탄소)       = [0.41, -0.20, 0.53, 0.28, -0.30]   ├ 비슷 (cosine ≈ 0.99)
  v(재생에너지) = [0.39,  0.05, 0.51, 0.30, -0.28]   ┘
  v(회의)       = [0.05,  0.31, 0.12, 0.45, 0.22]   ← 멀음 (cosine ≈ 0.30)
③ 그림 — 어떻게 학습되는가
핵심 가정: "비슷한 맥락에 자주 같이 나오는 단어는 의미가 비슷하다"

  문장 1: "당사는 [온실가스] 배출을 감축"
  문장 2: "당사는 [탄소] 배출을 감축"
  → [온실가스]와 [탄소]는 거의 같은 맥락 → 비슷한 좌표

수업 자료 표현: 단어 → 벡터 변환 (Week 5 BoW 그래도화)
④ 우리 연결
seed 30개에서 출발해 임베딩으로 "주변의 ESG 관련 단어"를 찾아 사전 확장(C-15). 발표 답안: "Embedding은 우리 corpus의 문맥을 학습해 ESG 단어 주변의 후보를 찾아내는 도구. seed가 너무 적어도 corpus 안에서 자연스럽게 확장 가능."
C-12 필수

fastText vs Word2Vec 단어 임베딩 모델 2종

① 한 줄 비유
Word2Vec = "단어를 통째로 학습", fastText = "단어를 글자 조각(subword)으로 쪼개 학습". 한국어처럼 어미 변화·합성어 많은 언어는 fastText가 유리.
② 작은 예시
"온실가스배출량" (사업보고서 신조어, 보통 띄어쓰기 안 함)

Word2Vec: "온실가스배출량" 통째 → 학습 못 본 단어면 벡터 없음 (OOV)
fastText: ["온실", "실가", "가스", "스배", "배출", "출량", ...]
          → 부분 단어 조합으로 벡터 생성 가능!
          → "온실가스" + "배출량"의 의미 일부를 상속
③ 표
특성Word2Vec (2013)fastText (2016)
학습 단위단어 전체단어 + subword (3-6 char n-gram)
OOV(미학습 단어)대처 불가부분 단어로 추정 가능
한국어 어미 변화"감축한다/감축했다/감축하지" 모두 다른 단어"감축" 공통 부분으로 유사도 잡힘
합성어"온실가스배출량" 미학습구성 요소로 의미 추정
학습 속도빠름느림 (subword 학습 추가)
④ 우리 연결
Li 2021은 word2vec(2018년 표준). 우리 가이드는 fastText 권장 — 한국어 + ESG 신조어/합성어 대응. 발표 답안: "한국어는 어미 변화가 많고 ESG 도메인엔 '온실가스배출' 같은 합성어가 흔해 fastText 선택. word2vec이면 미학습 합성어가 OOV가 되어 사전 확장 누락 위험."
C-13 필수

Cosine similarity 코사인 유사도

① 한 줄 비유
"두 벡터 사이의 각도". 같은 방향이면 1(매우 유사), 직각이면 0(무관), 반대면 -1. 크기(길이) 무시, 방향만 본다.
② 작은 예시
v(온실가스)   = [0.42, -0.18, 0.55]
v(탄소중립)   = [0.41, -0.17, 0.53]   → cosine ≈ 0.99 (같은 방향)
v(재생에너지) = [0.39,  0.05, 0.51]   → cosine ≈ 0.92 (거의 같은 방향)
v(회의)       = [0.05,  0.31, 0.12]   → cosine ≈ 0.31 (다른 방향)

공식: cos(A, B) = (A · B) / (|A| × |B|)
③ 그림
          ▲ 온실가스
          /
         /  θ ≈ 8°  → cos ≈ 0.99
        / /
   탄소중립
   ──────────────→  cos = 0 (90°, 무관)
                      회의
④ 우리 연결
두 곳에 등장: (1) C-14 θ sweep — seed 주변 후보 찾을 때 cosine ≥ θ인 단어 선택. (2) 기준문장 cosine — E/S/G 기준문장 TF-IDF 벡터 vs firm-year 문서 벡터의 cosine을 보조 feature로 (가이드 명시). 단 TF-IDF cosine은 단어 겹침 기반이라 embedding cosine과 다름.
C-14 필수

θ (threshold) sweep 임계값 스윕

① 한 줄 비유
"유사도 컷오프를 여러 값으로 바꿔보면서 사전 크기와 잡음 균형 찾기". 그물코를 촘촘하게(θ↑) vs 성기게(θ↓) 짜는 비교.
② 작은 예시
seed = "재생에너지"

fastText로 cos 상위 200단어 추출 후 θ 값별 결과:

θ = 0.90 (보수적):
  후보 = [탄소중립, 태양광]  → 2개. seed와 거의 같음. 확장 의미 없음
θ = 0.80 (균형):
  후보 = [탄소중립, 태양광, 저탄소, 풍력, 청정에너지, ...]  → 30개
  → 잡음 ~5%
θ = 0.70 (공격적):
  후보 = [위 + 에너지효율, 신재생, 대림에너지(잡음!), 한전(잡음!), ...]  → 150개
  → 잡음 ~25%, manual curation 부담↑
③ 그림 — Trade-off 곡선
      후보 수
      ▲
      │
 150  ─┤                                  ●
      │                              θ=0.70 (공격적)
      │                          노이즈 ↑↑
  30  ─┤                ●  ← sweet spot
      │           θ=0.80
   2  ─┤  ●
      │  θ=0.90  (확장 무의미)
      │
      └──────────────────────────────→
        0.90    0.85   0.80   0.75   0.70   θ값
                       ↑
                  Bao 2024 권장 영역
④ 우리 연결
심층가이드 결정박스 ⑤의 핵심. {0.70, 0.75, 0.80, 0.85, 0.90} 5개 값 sweep + 각 θ에서 cosine 상위 100단어 직접 검토. 발표 답안: "θ를 5개 값으로 sweep해 후보 수·잡음 비율 표 작성. 0.80에서 후보 30개·잡음 ~5%로 균형이 가장 좋아 채택. 회사명 N개를 manual 제외." 주의: 학술적으로는 0.75-0.85 범위가 표준 (Bao 2024).
C-15 필수

seed dictionary vs expanded dictionary 씨앗 사전 vs 확장 사전

① 한 줄 비유
seed = "교수가 정한 30개 출발점", expanded = "거기서 임베딩 + 검토로 넓힌 사전". 씨앗 30개를 심어 나무로 키워낸 것.
② 작은 예시
seed E (10개): 탄소, 온실가스, 탄소중립, 넷제로, 재생에너지,
              에너지, 전력, 폐기물, 재활용, 폐수

→ 각 seed에서 fastText cos 상위 단어 추출 (θ=0.80 적용)
→ 회사명·잡음 manual 제거
→ expanded E (예: 40개): seed 10개 +
       [태양광, 풍력, 청정에너지, 저탄소, 신재생, 배출량,
        감축, 절감, 친환경, ... ]
③ 그림 — 흐름
seed 30개 (교수 제공)
       ↓ fastText cos 계산
seed 주변 cos 상위 500개 단어
       ↓ θ 적용
θ를 통과한 단어 N개
       ↓ manual curation (회사명·일반명사 제거)
expanded dictionary 최종
       ↓ TF-IDF 합산
expanded_tfidf_score (주 비교 대상)
④ 우리 연결
01 overview md: "expanded dictionary는 seed 10개씩에서 출발해 단어 임베딩으로 후보를 넓힌 뒤 팀이 세운 기준으로 걸러낸 확장 사전". 핵심 메시지: 정답이 있는 사전이 아니라 팀이 직접 만들어 정당화해야 함. seed → expanded 점수 차이가 "텍스트 표현의 풍부함"을 보여줌.

D. 데이터 수집 lineage — DART API를 어떻게 두드리는가

OpenDART API는 한 번에 보고서를 못 가져오고 3단계 깔때기로 좁혀 들어갑니다. 식별자 3종이 헷갈리지 않게 그림으로 잡아두세요.

D-16 필수

OpenDART API 3단계 깔때기 CORPCODE → list → document

① 한 줄 비유
"전체 → 한 회사의 보고서 목록 → 그 회사의 특정 보고서 원문"으로 단계마다 좁혀 들어가는 깔때기.

목표: 삼성전자 2024 사업보고서 원문 받기

2단계: list.json (이 회사의 공시 목록)

   반환: [..., {report_nm: '사업보고서 (2024.12)', rcept_no: '20250311001085'}, ...]

   params: rcept_no=20250311001085

D-19 권장

Rate limit + 지수 backoff 호출 제한 + 재시도

① 한 줄 비유
Rate limit = "분당 호출 횟수 상한" (API 서버가 정함).
지수 backoff = "실패하면 점점 더 오래 기다리며 재시도" (1초 → 2초 → 4초).
② 작은 예시
OpenDART 제한: 분당 1,000회
우리 호출량: 381개 × ~3회 ≈ 1,143회

페이스 계산:
  분당 600회 목표 → 매 0.1초마다 1회 → time.sleep(0.1)
  전체 ~2시간 처리

실패 시 (5xx 또는 timeout):
  1차 재시도: 1초 후
  2차 재시도: 2초 후
  3차 재시도: 4초 후
  여전히 실패 → collection_log.csv에 사유 기록 후 패스
③ 그림 — 호출 페이스
0.0초 → 호출 1
0.1초 → 호출 2
0.2초 → 호출 3
  ...
60초 → 호출 600   ← 분당 600회 페이스 (제한 1000 안전 마진)

실패 흐름:
  요청 → 5xx → wait 1s → 재시도 → 5xx → wait 2s → 재시도 → 성공 ✓
  요청 → 5xx → wait 1s → ... → wait 4s → 5xx → collection_log 기록
④ 우리 연결
심층가이드 단계 4 "실무에서 챙겨야 할 것" 표 첫째·둘째 행. 발표 답안: "분당 600회 페이스로 ~2시간 처리. 일시 5xx 오류는 지수 backoff 3회 재시도 후 collection_log.csv에 사유 기록."

E. 통계 검증 — feature가 등급과 정말 연관되는가

회귀 들어가기 전 사전 검증. 학술적으로는 "Spearman + Mann-Whitney + 문장 직접 읽기 3종" 권장. 이 단계가 빠지면 평가표 "측정 validity"에서 감점.

E-20 필수

Spearman 순위상관 vs Pearson

① 한 줄 비유
Pearson = "키와 몸무게처럼 숫자값 그대로 상관", Spearman = "순위로만 비교" (1등이 점수도 1등인가). ESG 등급은 S/A+/A/B+/B/C/D 같은 서열변수이므로 Spearman이 자연스럽다.
② 작은 예시
기업    expanded_score(점수↓)   ESG_grade(높을수록↑)
  A         0.82                     S(6)
  B         0.65                     A(4)
  C         0.31                     B(2)

Pearson: 점수와 등급 숫자의 직선적 관계 (등급의 등간성 가정)
Spearman: 점수 순위 [1,2,3]과 등급 순위 [1,2,3]의 일치도
          → 모두 일치하면 ρ = 1.0
③ 표 — ρ 해석 기준
ρ 범위해석발표 시 언급
ρ < 0.10거의 없음"신호 미약, feature 재검토"
0.10 ≤ ρ < 0.20약한 신호"p가 작아도 효과는 작음, 추가 통제 필요"
0.20 ≤ ρ < 0.40중간"실용적 의미 있음, robustness 확인 필요"
ρ ≥ 0.40강함"드문 강신호, 단 cheap-talk 의심도 함께"
④ 우리 연결
심층가이드 단계 7, 04 평가표 "측정 validity". 발표 답안: "ESG 등급은 서열변수라 Pearson보다 Spearman이 자연스러움. 우리 결과 ρ=X.XX (p=XX)로 약한/중간 신호를 확인. 단 ρ가 작으면 통계 유의해도 효과는 작다고 해석."
E-21 권장

Mann-Whitney U test vs t-test

① 한 줄 비유
t-test = "두 그룹 평균이 다른가, 분포가 정규일 때", Mann-Whitney U = "분포 가정 없이 순위로 비교". 데이터 치우치면 후자가 안전.
② 작은 예시
질문: "A 이상 등급 그룹 vs B 이하 등급 그룹"의 expanded_score 평균이 다른가?

A 이상 그룹 (n=80):  점수 분포 [0.5, 0.6, 0.7, 0.8, 0.82, ...]
B 이하 그룹 (n=180): 점수 분포 [0.1, 0.2, 0.3, 0.4, 0.45, ...]

t-test: 평균 0.72 vs 0.35 → t=4.2, p<0.001
Mann-Whitney U: 두 그룹의 점수 순위 비교 → U=2150, p<0.001

→ 결과가 일치하면 robust. 다르면 분포 치우침이 t-test를 흔든 것.
③ 그림
정규 분포 (t-test OK):       치우친 분포 (Mann-Whitney 권장):
       ▲                              ▲
       │ ▲                            │█
       │█▲█                           │█▲
       │██▲██                         │██▲█
       │███████                       │█████████
④ 우리 연결
가이드는 "t-test 또는 Mann-Whitney" 둘 다 허용. 실무 권장: ESG 점수는 비대칭 분포 가능성이 있으므로 Mann-Whitney를 1차로, t-test를 보조로 보고하면 안전.
E-22 선택

p-value vs 효과 크기 statistical vs practical significance

① 한 줄 비유
p-value = "우연으로 보일 확률", 효과 크기 = "차이가 얼마나 큰가". p가 작아도 효과 크기가 작으면 실용적 의미는 약함.
② 작은 예시
케이스 A: ρ = 0.15, p = 0.003
  → 통계적으로 유의 (p<0.05) BUT 효과 약함 (ρ<0.20)
  → 해석: "양의 관계는 있으나 크기는 약함"

케이스 B: ρ = 0.45, p = 0.05
  → 통계적 경계 BUT 효과 강함 (ρ>0.40)
  → 해석: "강한 관계, 단 표본이 작아 p가 경계"

→ 둘 다 보고하기: ρ + p + 효과 해석
③ 그림 — 매트릭스
             p < 0.05      p >= 0.05
             ───────────  ───────────
효과 큼  │  강한 신호     │  표본 부족
효과 작음│  약한 신호     │  무관

→ 학생이 자주 하는 실수: p만 보고 "유의함!" → "강하지 않음" 누락
④ 우리 연결
가이드 04 "ρ가 0.10-0.20 수준이면 통계적으로 유의해도 효과는 약하다"에 직접. 발표 답안: "p와 효과 크기를 둘 다 보고. p<0.05라도 ρ가 0.15 수준이면 효과 약함을 명시."

F. 회귀 + 해석 — 모형 선택과 cheap-talk

feature가 검증을 통과했으면 회귀로 종합. 모형 3종 중 팀 질문에 맞춰 선택하고, cheap-talk 통제가 보고서의 학술 깊이를 결정.

F-23 필수

OLS / Ordered logit / Binary logit 회귀 모형 3종

① 한 줄 비유
OLS = "등급을 점수로 바꿔 직선 회귀" (등간성 가정).
Ordered logit = "등급의 순서만 가정" (가장 학술적).
Binary logit = "A 이상 vs 미만으로 2분류" (가장 직관적).
② 작은 예시
가정 차이:
  OLS:           A+에서 A로 떨어지는 차이 = B에서 C로 떨어지는 차이
                 (등간성 — 사실 무리)
  Ordered logit: 순서만 보존. A+→A 차이와 B→C 차이가 다른 크기여도 OK
  Binary logit:  A 이상 = 1, B 이하 = 0으로 단순화

회귀식 예 (OLS):
  esg_grade_num = β0 + β1·expanded_tfidf_score + β2·total_word_count + ε
③ 표
모형가정장점단점언제
OLS등간성해석 직관적등간성 무리baseline 빠르게
Ordered logit순서만가장 정확해석 까다로움주 모형 (권장)
Binary logit2분류odds ratio 직관등급 차이 손실"우수 vs 보통" 질문
④ 우리 연결
심층가이드 결정박스 ⑧. 발표 답안 권장 조합: "OLS를 baseline, ordered logit으로 robustness 확인. β1의 방향·유의성이 두 모형에서 일치하면 결과 안정적."
F-24 권장

다중공선성 Multicollinearity

① 한 줄 비유
"비슷한 두 변수를 한 회귀에 동시에 넣으면 계수가 망가지는 현상". 두 명에게 같은 일을 시키면 누구 공인지 모르는 것과 같음.
② 작은 예시
나쁜 회귀식:
  grade ~ seed_tfidf_score + expanded_tfidf_score + word_count
                ↑                  ↑
            상관 ≈ 0.95 (거의 같은 정보!)
  → β1, β2 계수가 둘 다 불안정해짐. 부호 뒤집힐 수도

좋은 분리 (M1a vs M1b):
  M1a: grade ~ seed_tfidf_score + word_count
  M1b: grade ~ expanded_tfidf_score + word_count
  → 각각 별도 모델로 비교
③ 그림
seed_score와 expanded_score는 큰 부분이 겹친다:

  ┌── seed ──┐
  │  단어 30  │
  └──────────┘
       ↓ 확장
  ┌── expanded ──────────────────┐
  │ 단어 30 (seed)  +  확장 단어 N │  → seed의 모든 정보 포함
  └──────────────────────────────┘

  두 점수를 동시 회귀 = 같은 정보를 두 번 넣은 셈
④ 우리 연결
심층가이드 6단계 "다중공선성 주의" 박스. 발표 답안: "seed_score와 expanded_score 상관이 매우 높아 동시 회귀 금지. M1a/M1b로 분리하고 각 효과 크기를 비교."
F-25 필수

cheap-talk 가설 + total_word_count 통제 disclosure verbosity control

① 한 줄 비유
"많이 쓰기만 해도 점수 오르는 함정". 진짜 ESG 성과가 아니라 그냥 사업보고서가 길고 장황해서 점수가 높은 것일 수 있음. 영어 학술용어로는 disclosure verbosity 또는 "cheap-talk".
② 작은 예시
회사 A (실제 ESG 우수): 짧고 구체적 보고서, 1,000 단어 중 ESG 단어 50개
회사 B (말만 많음):      길고 장황한 보고서, 5,000 단어 중 ESG 단어 200개

TF-IDF 합산만 보면:
  A: 50 × IDF = X
  B: 200 × IDF = 4X   ← B가 더 우수해 보임!

word_count 통제 회귀:
  M1: grade ~ expanded_score          → β1 큼
  M2: grade ~ expanded_score + word_count  → β1 줄어듦?
  M2에서도 β1 유의·양수 → 진짜 ESG 신호
  M2에서 β1 ≈ 0       → cheap-talk 가설 강화
③ 그림 — 검증 회귀식 3종
M1 (단순):     grade ~ expanded_score
M2 (통제):     grade ~ expanded_score + word_count
M3 (강화):     grade ~ expanded_score + word_count + 업종 더미

읽는 법:
  M1→M2에서 β1이 거의 안 바뀌면 → 진짜 ESG 신호
  M1→M2에서 β1이 크게 감소하면 → cheap-talk 의심
  M3에서 β1 여전히 유의 → 업종 차이로도 설명 안 됨
④ 우리 연결
심층가이드 6단계 "cheap-talk 통제변수" + 8단계 회귀식 해석 + 04 평가표 "cheap-talk 가능성"에 모두 등장. 학술 깊이의 핵심 포인트. 발표 답안: "M1에서 expanded_score 계수가 유의했으나 word_count를 통제한 M2에서 β1이 30% 감소. 신호의 일부는 cheap-talk(공시 장황함)에서 오는 것으로 해석. 단 통제 후에도 β1이 양수·유의로 남아 진짜 ESG 신호가 부분적으로 존재."
