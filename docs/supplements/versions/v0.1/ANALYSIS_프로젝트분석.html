ESG DART 기말 프로젝트 — 프로젝트 분석

교수님 가이드 4개 md를 한 줄에 꿰어 "무엇을·왜·어떻게"를 보여줍니다. 비정형데이터 처리와 데이터 마이닝 관점에서 수업과 어떻게 연결되는지를 비전공자 시점으로 풀었습니다.

대상 표본 127기업 × 3년 = 381 firm-years 핵심 방법 TF-IDF · fastText · cosine · 회귀 가이드 출처 01~04 md 버전 v0.1

한 페이지 요약 — 입력에서 결과까지

DART 사업보고서의 "말"을 숫자로 바꿔서, 그 숫자가 KCGS가 매긴 ESG 등급과 어떤 관계가 있는지 보는 프로젝트입니다. 정답을 맞히는 과제가 아니라 선택을 정당화하는 과제입니다.

학원에서 학생이 시험을 보면, 그 학생의 답안지(공시 사업보고서)에 적은 글에서 ESG와 관련된 단어가 얼마나 자주, 얼마나 의미있게 등장하는지를 점수로 만든 뒤, 그 점수가 외부 평가기관(KCGS)이 매긴 등급과 같이 움직이는지를 보는 것입니다. 같이 움직인다고 해서 "글이 등급을 결정했다"는 게 아니라 "글과 등급 사이에 패턴이 있다"는 것을 데이터로 확인합니다.

이 프로젝트가 진짜로 묻는 것

교수님 가이드는 "정답 결과값을 재현하는 과제가 아니다"라고 명시했습니다. 그럼 무엇을 묻는 걸까요?

🎯 한 문장

"공시 언어로 만든 측정값이 KCGS ESG 등급과 어떤 패턴으로 연결되는가?"

등급을 맞히려는 분류 문제가 아닙니다. 텍스트에서 뽑은 숫자가 무엇을 포착하는지를 확인하고, 외부 평가와 어떻게 연결되는지를 해석하는 과제입니다.

학생의 자기소개서 vs. 학교생활기록부 평점의 관계를 보는 것과 비슷합니다. 자기소개서에 "리더십" 단어를 많이 쓴 학생이 실제로 평점이 높을까? 같이 움직인다면 그건 자소서가 평점을 결정한 게 아니라, 자소서 표현과 평가 사이에 일종의 "기호적 일치"가 있다는 뜻입니다. 우리는 이 일치를 데이터로 보는 것.

이 질문을 묻는 이유 — 비정형 데이터 분석의 핵심 가치

📜 비정형 데이터 = 정형 데이터로 못 보는 것

KCGS 등급(S/A+/A/B+/B/C/D)은 이미 깔끔한 정형 데이터. 하지만 그 뒤에는 분석가들이 본 사업보고서의 표현, 문장의 톤, 강조점이 있습니다. 이걸 텍스트에서 직접 끌어오면 등급으로 환원되지 않은 차원이 보입니다.

🔍 데이터 마이닝 = 패턴을 찾는 것

381 firm-years × 수십만 단어에서 "어떤 단어가 등급과 같이 움직이나" 같은 약한 신호 패턴을 찾는 것이 데이터 마이닝의 본질. 강한 결정론적 규칙이 아니라 통계적 연관성입니다.

세 가지 조심할 것 (해석 가드레일)

🔴 단정하지 말 것 #1

"이 표본에서는 ESG 단어가 강한 기업일수록 등급도 높았다"까지만. "공시가 등급을 결정했다"는 인과 단정 금지.

🔴 단정하지 말 것 #2

사업보고서 표현 ≠ 실제 ESG 성과. 어떤 기업은 잘해서 많이 쓰고, 어떤 기업은 설명이 필요한 이슈가 있어서 많이 씁니다 (cheap-talk).

🔴 단정하지 말 것 #3

381행은 수업용 pilot 표본. KCGS 전체 기업의 무작위 대표표본이 아닙니다.

식별자 3종의 관계 — "깔때기" 비유

OpenDART는 깔때기 모양입니다. 위에서 stock_code(종목코드)가 들어가면 → 중간에서 corp_code(회사 내부 코드)로 변환되고 → 마지막에 그 회사의 rcept_no(접수번호) 목록이 나옵니다. 우리는 그중 "사업보고서 (2024.12)" 행을 골라야 합니다.
📌 timing 규칙 (절대값)
esg_year = fiscal_year + 1

예시:
  2025년 KCGS 등급  ←→  2024.12 사업보고서
  2024년 KCGS 등급  ←→  2023.12 사업보고서

이유: KCGS는 직전 사업연도 정보를 보고 등급을 매기기 때문. 이 규칙이 흔들리면 회귀 결과가 모두 흔들립니다.

텍스트를 숫자로 — 이번 학기 핵심 마이닝 기법

"당사는 온실가스 배출량을 8.3% 감축…" 같은 문장을 어떻게 점수로 바꿀까요? 4가지 방법을 단계적으로 쌓습니다.

방법 ① — TF-IDF (seed score)

TF-IDF가 뭔가?

TF(Term Frequency, 단어 빈도) × IDF(Inverse Document Frequency, 역문서 빈도). 한 단어가 이 문서에서는 자주 나오는데 다른 문서에서는 잘 안 나올수록 점수가 큽니다.

학원에서 "수학"이라는 단어는 모든 학생 자소서에 나옵니다 (IDF 낮음 → 점수 낮음). 하지만 "올림피아드"는 특정 학생 자소서에만 자주 나오면 그 학생의 차별점이 됩니다 (IDF 높음 → 점수 높음).
🧮 seed score 계산법
seed 30개 = E 10개 + S 10개 + G 10개

E seed = [탄소, 온실가스, 탄소중립, 넷제로, 재생에너지,
         에너지, 전력, 폐기물, 재활용, 폐수]

E seed score (한 firm-year) = TF-IDF(탄소) + TF-IDF(온실가스) + ...

같은 방식으로 S score, G score 계산.

방법 ② — fastText 확장 사전 (expanded score)

왜 확장하나?

seed 30개는 출발점일 뿐. 실제 보고서에는 저탄소, 탄소포집, RE100 같은 단어도 ESG와 관련됩니다. 이런 의미적으로 비슷한 단어를 word embedding으로 찾아냅니다.

seed가 "온실가스, 재생에너지"라면, fastText는 corpus 안에서 이 두 단어와 비슷한 자리에 등장하는 다른 단어들을 찾아 점수(cosine similarity)를 매깁니다. "탄소중립"(0.92), "배출권"(0.88), "저탄소"(0.85)처럼. 우리는 일정 점수 이상(threshold θ)인 단어를 후보로 삼습니다.

θ sweep — "감"이 아니라 "데이터로" 정하기

🟡 권고 — θ를 여러 값으로 시도
θ후보 수잡음 비율특징
0.70~500개높음 (회사명·일반명사 多)너무 넓게 잡아 잡음 증가
0.80~150개중간균형점 후보
0.85~50개낮음안전하지만 진짜 ESG 단어도 놓침
0.90~10개매우 낮음seed에 거의 추가 없음

각 θ별로 상위 100단어를 직접 눈으로 검토해서 잡음 비율을 표로 정리. 최종 θ를 정할 때 회사명·일반명사를 직접 걸러냅니다.

🔴 위반 금지 — fastText 결과를 검토 없이 그대로 쓰기

fastText는 "비슷한 자리"를 보는 거라 의미가 비슷하지 않은 단어도 후보에 들어옵니다. 예: "대림에너지"(회사명) cosine 0.78 — 사람이 보고 제외해야 합니다. (Bao et al. 2024 표준 절차)

방법 ③ — 기준 문장 cosine similarity (보조)

아이디어

교수님이 제시한 E·S·G 기준 문장과 우리 corpus의 firm-year 문서를 둘 다 TF-IDF 벡터로 만들어, 벡터의 방향이 얼마나 비슷한지를 cosine으로 측정.

차원기준 문장
E온실가스 배출 감축, 재생에너지 사용, 탄소중립 목표와 실행 계획
S산업안전, 근로자 보호, 공급망 책임, 고객과 지역사회에 대한 사회적 책임
G이사회 독립성, 감사위원회, 내부통제, 주주권리와 윤리경영
한계 — cosine은 "단어 겹침"만 본다 TF-IDF cosine은 의미 임베딩이 아니라 표면 단어 겹침에 가까운 측정. dense embedding으로 확장하면 [심화] 알파 분석 후보. (03 md)

방법 ④ — 양으로 통제 (cheap-talk control)

📊 total_word_count 통제 변수

어떤 기업은 그냥 보고서가 길어서 ESG 단어도 많이 등장할 수 있습니다 (= 장황함, cheap-talk). 회귀에 total_word_count를 같이 넣으면, "양을 통제한 후에도 점수가 등급과 연결되는가"를 볼 수 있습니다.

검증과 회귀 — 점수가 등급과 정말 연결되나?

feature를 만들었다고 끝이 아닙니다. 회귀로 바로 가지 말고, 먼저 통계 검증으로 점수와 등급의 관계를 확인하는 것이 04 md의 평가표 항목입니다.

1단계 검증 — Spearman 순위상관

🟡 권고 — Spearman ρ를 먼저
  • 왜 Spearman? ESG 등급은 D~A+의 서열변수. 등간성(A+→A 차이 = B→C 차이)이 아닙니다. 순위가 같이 움직이는지를 보는 게 자연스러움.
  • 판단 기준: |ρ|가 0.10~0.20이면 통계적으로 유의해도 약한 연관. p-value만 보지 말 것.
| feature              | Spearman ρ | p-value | 해석                  |
| expanded_tfidf_score | 0.15       | 0.004   | 약한 양의 관계        |
| seed_tfidf_score     | 0.03       | 0.55    | 뚜렷한 관계 없음      |

2단계 검증 — 등급 그룹 간 평균 차이

🟡 권고 — t-test 또는 Mann-Whitney

"상위 등급(A 이상) 그룹"과 "하위 등급(B+ 이하) 그룹"의 평균 점수가 다른가? 분포가 정규성에 가까우면 t-test, 그렇지 않거나 표본이 작으면 Mann-Whitney U. Spearman이 못 잡는 그룹별 차이를 보완합니다.

3단계 — 회귀모형 선택 (옵션 A·B·C)

옵션 A — OLS

직관적 baseline

esg_grade_num = β0 + β1·score
              + β2·word_count + ε

장점 해석 쉬움. 한계 등급의 등간성을 가정 (엄밀히는 깨짐).

옵션 B — Ordered logit

등급 순서 보존

ordered_logit(grade)
  = β1·score + β2·word_count

장점 서열변수에 가장 잘 맞음. 한계 계수 해석이 어려움, 표본 작으면 불안정.

옵션 C — Binary logit

"A 이상 여부"로 단순화

P(grade ≥ A)
  = logistic(β0 + β1·score + ...)

장점 odds ratio로 해석 쉬움. 한계 등급 정보 축소.

💡 선택 가이드 정답은 없습니다. 팀의 질문에 맞는 모형을 고르고 버린 대안의 한계를 보고서에 함께 적습니다. (예: "우리는 ordered logit을 선택했다 — 등급이 서열변수라서. OLS는 등간성 가정이 깨지고, binary logit은 등급 정보를 잃기 때문에 보조로만 보고했다.")

해석 가드레일 (cheap-talk 가설)

결과 패턴조심스러운 해석
β1 양수 + 유의 + word_count 통제 후에도 유지텍스트 신호가 단순 장황함 이상
β1 양수 + 유의 + word_count 통제 후 0에 가까워짐신호의 상당 부분이 disclosure verbosity (cheap-talk 가설 강해짐)
부호 불안정, 표본 작음다중공선성 또는 표본 한계
🔴 다중공선성 위험 seed_tfidf와 expanded_tfidf를 동시에 회귀에 넣지 마세요. 두 변수가 강하게 상관되어 계수가 불안정해집니다. 별도 모델로 비교합니다.

강의에는 없지만 이 프로젝트에 추가되는 것

📌 회귀분석 (OLS/ordered/binary logit)

통계학 기초에서 다뤄지지만, 이번 학기 NLP 수업의 정식 토픽은 아님. statsmodels 라이브러리로 처리합니다.

📌 검증 통계 (Spearman, Mann-Whitney)

비모수 통계 검증. scipy.stats로 한 줄 호출 가능. p-value보다 효과 크기(ρ)에 집중.

위반 금지 + 평가 기준

04 md 평가표를 비전공자가 봐도 알 수 있게 풀었습니다. 보고서 작성 시 옆에 두세요.

🔴 절대 어기면 안 되는 것 (위반 시 감점)

위반 #1 — 회사명 join

company_name으로 merge하면 사명변경·분할로 silent 오매칭. stock_code 기준.

위반 #2 — timing 어김

esg_year = fiscal_year + 1 어기면 등급-보고서 시점이 어긋나서 회귀 결과가 무의미.

위반 #3 — 가짜 0

수집 실패 행을 텍스트 점수 0으로 채우면 ❌. collection_log에 이유 기록 후 분석에서 drop.

위반 #4 — API key 노출

노트북 출력·코드에 crtfc_key 직접 보이면 ❌. .env만 사용.

위반 #5 — fastText 검토 없이

θ만 정하고 후보 단어를 사람이 안 본 채 expanded dict 확정 ❌. 회사명·일반명사 직접 제거.

위반 #6 — 다중공선성

seed_tfidf와 expanded_tfidf를 한 모델에 동시 투입 ❌. 별도 모델로 비교.

위반 #7 — 인과 단정

"공시가 등급을 결정한다"식 표현 ❌. "이 표본에서는 ~ 경향이 있었다"로 조심스럽게.

위반 #8 — MCP 결과 무검증

MCP가 만든 passage 표를 그대로 분석에 넣으면 ❌. Python 코드로 재현·검증 필요.
